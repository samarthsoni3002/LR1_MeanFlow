# LR1_MeanFlow

A small PyTorch implementation of a **MeanFlow-style pixel-space one-step image generation overfitting experiment**.

The project downloads Imagenette, selects only three fixed training images, trains a small conditional U-Net to overfit those images at **256x256 resolution**, and saves the training outputs automatically.

The main entry point is:

```bash
python train_overfit.py
```

Running this script handles the full experiment: data download, preprocessing, model training, checkpoint saving, loss-curve plotting, and original-vs-generated image visualization.

## What This Experiment Does

`train_overfit.py` performs the following steps:

1. Creates `outputs/` and `checkpoints/` folders if they do not already exist.
2. Uses GPU if CUDA is available; otherwise uses CPU.
3. Downloads and extracts the `imagenette2-320` dataset if it is not already present.
4. Resizes and center-crops images to `256 x 256`.
5. Selects only **three images**, one from each of three different classes.
6. Remaps the selected labels to `0, 1, 2`.
7. Trains `MeanFlowUNet` on only those three fixed images.
8. Uses the MeanFlow JVP objective without auxiliary endpoint reconstruction loss.
9. Saves the trained checkpoint.
10. Saves the training loss curve.
11. Generates samples from noise using the one-step MeanFlow sampler.
12. Saves an original-vs-generated comparison image.

## MeanFlow Objective Used

The image/noise interpolation path is:

```python
z_t = (1 - t) * x + t * eps
```

where:

* `x` is the clean image,
* `eps` is Gaussian noise,
* `t = 0` corresponds to the image,
* `t = 1` corresponds to noise.

The instantaneous velocity along this path is:

```python
v = eps - x
```

The model predicts the average velocity `u(z_t, r, t)`. The JVP-based MeanFlow target is:

```python
u_tgt = v - (t - r) * dudt
```

The final training loss is:

```python
loss = ((u - u_tgt.detach()) ** 2).mean()
```

No endpoint reconstruction loss is used in the final 256x256 overfit run.

During generation, the sampler starts from noise and performs one model evaluation:

```python
x_gen = z - u(z, r=0, t=1)
```

## 256x256 Setup

The final corrected setup uses:

* Dataset: `imagenette2-320`
* Image transform: `Resize(256)` + `CenterCrop(256)`
* Training image shape: `[3, 3, 256, 256]`
* Generated image shape: `[3, 3, 256, 256]`
* Number of training examples: `3`
* Model: small conditional U-Net
* Model capacity:

  * `base_channels = 64`
  * `time_emb_dim = 128`
* Input enhancement: coordinate channels added to the image/noise input
* Objective: pure MeanFlow JVP loss
* Sampler: one-step MeanFlow sampler

## Reproducing the Experiment

### 1. Clone the repository

```bash
git clone https://github.com/samarthsoni3002/LR1_MeanFlow.git
cd LR1_MeanFlow
```

### 2. Create a virtual environment

#### Windows

```bash
python -m venv .venv
.venv\Scripts\activate
```

#### macOS / Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

If you want GPU acceleration, make sure your installed `torch` and `torchvision` versions match your CUDA version. If the default install does not detect your GPU, install PyTorch using the command recommended on the official PyTorch installation page for your CUDA version.

### 4. Run the experiment

```bash
python train_overfit.py
```

That is the main command needed to reproduce the three-image 256x256 overfit run.

## Notes on the 256x256 Fix

The original implementation used a smaller image resolution. I updated the pipeline to correctly run the required 256x256 overfit experiment.

The main fixes were:

1. Changed the dataset source from `imagenette2-160` to `imagenette2-320`.
2. Changed preprocessing from `32 x 32` resizing to `256 x 256` resizing and center cropping.
3. Removed hardcoded `32 x 32` sampling noise.
4. Generated sampling noise using the real batch shape.
5. Increased model capacity for 256x256 memorization.
6. Added coordinate channels to help the model memorize fixed spatial layouts.
7. Kept the final loss as the pure MeanFlow JVP objective without endpoint reconstruction loss.
8. Added shape checks to verify that the original and generated tensors are both `[3, 3, 256, 256]`.
